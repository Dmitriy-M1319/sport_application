Требования к мобильному приложению для учёта тренировок

Функциональные требования

1. Управление списком тренировок

* Пользователь должен иметь возможность создавать новую тренировку с указанием даты, времени и типа (силовая, кардио, растяжка и др.)
* Типы возможных тренировок: силовая тренировка, кардио тренировка, занятие йогой, аэробная тренировка, ходьба
* Пользователь должен иметь возможность просматривать список всех тренировок в хронологическом порядке (по порядку их создания от новых к старым)
* В списке в ячейке тренировки отображаются: название тренировки, категория тренировки, дата создания
* Пользователь должен иметь возможность редактировать параметры существующей тренировки (название тренировки, тип тренировки, список упражнений)
* Пользователь должен иметь возможность удалять тренировку из списка
* Список тренировок должен поддерживать фильтрацию по типу, дате и диапазону дат
* Список тренировок должен поддерживать поиск по названию или типу тренировки

2. Управление упражнениями внутри тренировки

* Пользователь должен иметь возможность добавлять упражнения в тренировку из справочника или вручную
* Для каждого упражнения должна фиксироваться следующая информация: название, количество подходов, количество повторений, рабочий вес
* Пользователь должен иметь возможность изменять порядок упражнений в списке (перетаскиванием)
* Пользователь должен иметь возможность дублировать упражнение внутри тренировки
* Пользователь должен иметь возможность удалять упражнение из тренировки

3. Справочник упражнений

* Приложение должно содержать встроенный список упражнений с разбивкой по группам мышц (является глобальным для всех пользователей по умолчанию)
* Пользователь должен иметь возможность добавлять собственные упражнения в справочник
* Пользователь должен иметь возможность редактировать и удалять пользовательские упражнения
* Информация о тренировке отображается: название, количество подходов, рабочий вес - в карточке упражнения, полное описание - на отдельной странице
* Справочник должен поддерживать фильтрацию по группе мышц и поиск по названию

4. Статистика и прогресс

* Приложение должно отображать список личных рекордов по каждому упражнению (максимальный вес, максимальное количество повторений)
* Приложение должно отображать историю результатов по выбранному упражнению в виде списка с сортировкой по дате (отображаются количество повторений на рабочий вес как единая метрика)
* Приложение должно подсчитывать суммарный объём нагрузки за тренировку (сумма: вес × повторения × подходы). Подсчитанный объем сохраняется для выбранной тренировки и отображается в полном описании тренировки на отдельной странице

5. Авторизация и профиль

* Пользователь должен иметь возможность зарегистрироваться и войти в приложение
* Регистрация основывается на использовании номера телефона с одноразовым кодом входа по СМС. Сессия авторизации сохраняется на устройстве 3 дня. Подтверждать учетную запись не требуется
* Пользователь должен иметь возможность указать персональные данные: имя, возраст, вес, рост. Ввод осуществляется при первом входе в приложение и может в дальнейшем редактироваться на странице просмотра персональных данных
* Приложение должно поддерживать выход из учётной записи. При утрате номера телефона можно обратиться в техподдержку

3 =============== Последний вариант требований ============
Требования к мобильному приложению для учёта тренировок

2. Функциональные требования
2.1 Управление списком тренировок

* Пользователь должен иметь возможность создавать новую тренировку с указанием даты, времени и типа.
* Название тренировки задаётся вручную пользователем (от 1 до 100 символов).
* Типы возможных тренировок: силовая тренировка, кардио тренировка, занятие йогой, аэробная тренировка, ходьба.
* Пользователь должен иметь возможность просматривать список всех тренировок в хронологическом порядке (от новых к старым по дате создания).
* В списке в ячейке тренировки отображаются: название тренировки, категория (тип) тренировки, дата создания.
* Пользователь должен иметь возможность редактировать параметры существующей тренировки (название тренировки, тип тренировки, список упражнений).
* Пользователь должен иметь возможность удалять тренировку из списка. При удалении отображается диалог подтверждения.
* Список тренировок должен поддерживать фильтрацию по типу тренировки и диапазону дат создания (пользователь выбирает начальную и конечную дату).
* Список тренировок должен поддерживать поиск по названию или типу тренировки.

Пустые состояния:

* Если у пользователя нет созданных тренировок, отображается заглушка с текстом «У вас пока нет тренировок» и кнопкой «Создать первую тренировку».
* Если поиск или фильтрация не дали результатов, отображается заглушка с текстом «Ничего не найдено. Попробуйте изменить параметры поиска».

2.2 Управление упражнениями внутри тренировки

* Пользователь должен иметь возможность добавлять упражнения в тренировку из справочника или вручную. Максимальное количество упражнений в одной тренировке — 50.
* Добавление из справочника: при нажатии кнопки «Добавить упражнение» открывается экран справочника с возможностью поиска и фильтрации по группе мышц. Пользователь выбирает одно или несколько упражнений и подтверждает добавление. Выбранные упражнения добавляются в конец списка тренировки.
* Добавление вручную: при нажатии кнопки «Создать своё упражнение» пользователь вводит название и описание. Упражнение сохраняется в пользовательский справочник и добавляется в тренировку.
 
Для каждого упражнения должна фиксироваться следующая информация:

* Название (от 1 до 100 символов).
* Количество подходов (целое число от 1 до 99).
* Количество повторений (целое число от 1 до 999).
* Рабочий вес в кг (число от 0 до 999.9, с шагом 0.1 кг; значение 0 допустимо для упражнений с собственным весом).


* Пользователь должен иметь возможность изменять порядок упражнений в списке (перетаскиванием).
* Пользователь должен иметь возможность дублировать упражнение внутри тренировки (создаётся копия с теми же параметрами).
* Пользователь должен иметь возможность удалять упражнение из тренировки.

2.3 Справочник упражнений

* Приложение должно содержать встроенный (глобальный) список упражнений с разбивкой по группам мышц.
* Группы мышц: грудные, спина, плечи, бицепс, трицепс, предплечья, пресс, ягодицы, квадрицепс, бицепс бедра, икры, всё тело.
* Пользователь должен иметь возможность добавлять собственные упражнения в справочник (название от 1 до 100 символов, описание до 500 символов, выбор группы мышц).
* Пользователь должен иметь возможность редактировать и удалять только свои (пользовательские) упражнения. Глобальные упражнения редактированию и удалению не подлежат.
* Если пользовательское упражнение используется в существующей тренировке, при попытке удаления отображается предупреждение: «Это упражнение используется в N тренировках. При удалении оно будет удалено из всех тренировок. Продолжить?»
* Информация об упражнении отображается: в карточке — название, количество подходов, рабочий вес; на отдельной странице — полное описание, группа мышц, история результатов.
* Справочник должен поддерживать фильтрацию по группе мышц и поиск по названию.

Пустые состояния:

* Если в результате поиска/фильтрации не найдено упражнений, отображается заглушка «Упражнения не найдены».

2.4 Статистика и прогресс

* Приложение должно отображать список личных рекордов по каждому упражнению:

- Максимальный вес (наибольшее значение рабочего веса среди всех тренировок).
- Максимальное количество повторений (наибольшее значение повторений в одном подходе среди всех тренировок).

* Личные рекорды обновляются автоматически при сохранении тренировки. При удалении тренировки рекорды пересчитываются на основании оставшихся данных.
* Приложение должно отображать историю результатов по выбранному упражнению в виде списка с сортировкой по дате. Каждая запись содержит: дату тренировки, рабочий вес, количество повторений, количество подходов и расчётный объём (вес × повторения × подходы).
* Приложение должно подсчитывать суммарный объём нагрузки за тренировку по формуле: Σ (вес × повторения × подходы) по всем упражнениям тренировки. Подсчитанный объём сохраняется для выбранной тренировки и отображается в полном описании тренировки на отдельной странице.

2.5 Авторизация и профиль

* Пользователь должен иметь возможность зарегистрироваться и войти в приложение.
* Регистрация основывается на использовании номера телефона с одноразовым кодом входа по СМС. Подтверждать учётную запись не требуется.
* Код СМС: 6-значный числовой код, действителен в течение 5 минут. Повторный запрос кода возможен не ранее чем через 60 секунд. Максимум 5 запросов кода в течение 1 часа на один номер телефона (rate-limiting). При превышении лимита отображается сообщение «Слишком много попыток. Повторите через N минут».

Обработка ошибок авторизации:

* Неверный код: отображается сообщение «Неверный код. Осталось N попыток». После 5 неудачных попыток ввода кода текущий код аннулируется, пользователю предлагается запросить новый.
* Истёкший код: отображается сообщение «Срок действия кода истёк. Запросите новый код».
* Ошибка сети: отображается сообщение «Нет соединения с сервером. Проверьте подключение к интернету» с кнопкой «Повторить».

* Сессия авторизации сохраняется на устройстве 3 дня с момента последней активности. По истечении срока сессия завершается автоматически: пользователь перенаправляется на экран входа с возможностью запросить новый СМС-код. При активном использовании приложения сессия продлевается.
* Пользователь должен иметь возможность указать персональные данные: имя (от 1 до 50 символов), возраст (целое число от 14 до 120), вес в кг (число от 20.0 до 350.0, шаг 0.1), рост в см (целое число от 100 до 280). Ввод осуществляется при первом входе в приложение и может в дальнейшем редактироваться на странице просмотра персональных данных.
* Приложение должно поддерживать выход из учётной записи. При утрате номера телефона можно обратиться в техподдержку (в приложении указан email поддержки и ссылка на форму обратной связи).


3. Нефункциональные требования
3.1 Производительность

* Время загрузки приложения (холодный старт): не более 3 секунд на устройстве среднего класса.
* Время отклика интерфейса на действия пользователя (переход между экранами, открытие списка): не более 500 мс.
* Время отклика серверного API: не более 500 мс для 95% запросов.

3.2 Оффлайн-режим

* Приложение должно поддерживать работу в оффлайн-режиме: просмотр ранее загруженных тренировок, создание и редактирование тренировок.
* При восстановлении соединения данные автоматически синхронизируются с сервером.
* При наличии конфликтов (данные изменены на нескольких устройствах) приоритет отдаётся последнему изменению по времени.

3.3 Синхронизация между устройствами

* Данные пользователя (тренировки, справочник, профиль, рекорды) синхронизируются между устройствами при входе под одной учётной записью.
* Синхронизация выполняется автоматически при наличии сетевого подключения.

3.4 Безопасность

* Все соединения между приложением и сервером осуществляются по HTTPS.
* СМС-коды хранятся в хэшированном виде на сервере.
* Персональные данные пользователей хранятся в зашифрованном виде.
